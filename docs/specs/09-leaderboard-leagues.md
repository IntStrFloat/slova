# 09 — Лидерборды и лиги

Статус: утверждено · Обновлено: 2026-06-26

Серверо-авторитетно (backend [11](11-profile-cloudsave-backend.md)). Клиент — `features/leaderboard` (+ `features/leagues`), кэш в `leaderboard.v1`, offline-first очередь сабмитов.

## Метрика ранжирования

- Основная: **суммарный счёт прогресса** (очки за пройденные уровни/слова/бонусы) и/или число пройденных уровней. Точную формулу задаёт `core/config` и **бэкенд пересчитывает** из подтверждённого прогресса (клиентский score недоверенный).
- Вторичная для недели: прирост за неделю (weekly best/Δ).

## Типы таблиц

| Тип | Сброс | Источник |
|---|---|---|
| Глобальная (all-time) | — | накопленный прогресс |
| Недельная | пн 00:00 UTC | прирост за неделю |
| Лига/дивизион | неделя | группа ~30–50 игроков, промоушн/демоушн |

## Лиги (`features/leagues`)

- Дивизионы: Бронза → Серебро → Золото → Платина → Алмаз.
- Неделя: топ-K дивизиона повышаются, низ-K понижаются; середина остаётся. Раздача наград (монеты/кристаллы) по местам.
- Назначение в дивизион/группу — на бэкенде (балансировка по уровню игрока).

## Клиентский поток (offline-first)

```
прогресс уровня → submit в очередь (leaderboard.v1) → отправка при сети
GET /api/leaderboard?scope=global|weekly|league → снапшот (топ-100 + я ±соседи)
рендер: ник+tag+аватар, моя строка подсвечена; pull-to-refresh
```

## Античит

- Сервер принимает **подтверждённый прогресс** (см. ревалидацию [11]); не доверяет присланному счёту.
- Rate-limit на сабмиты (nginx + app-key, как в blockblast). Санити-лимиты (макс. уровней/прирост за период).
- Аномалии помечаются (shadow), не попадают в публичную ленту.

## UI

Экран `/leaderboard` с вкладками Global/Weekly/League; карточка лиги (текущий дивизион, до промоушна/демоушна N мест, таймер недели); пустое/оффлайн-состояние; «привяжи аккаунт, чтобы соревноваться» если гость [11].

## Тесты

Очередь/ретрай сабмитов, корректность снапшота (моя позиция, соседи), переход недели, промоушн/демоушн (бэкенд-тесты).
