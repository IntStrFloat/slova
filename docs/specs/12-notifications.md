# 12 — Push-уведомления (RuStore)

Статус: утверждено · Обновлено: 2026-06-26

SDK: `react-native-rustore-push` (GitFlic), подключение Expo config-plugin в `plugins/withRuStorePush.js` (зеркало blockblast). За интерфейсом + флаг `FEATURES.push`.

## Поток

- **Soft-ask** после онбординга/первой победы (не на холодном старте). Только при согласии — запрос системного разрешения и регистрация токена.
- Приём токена → отправка на backend (`POST /api/push/register { token, userId }`) для серверных кампаний.
- Обработка foreground/background/cold-start тапов → диплинк на нужный экран (`slova://daily`, `slova://wheel`, `slova://events`).

## Кампании (ретеншн)

| Триггер | Текст (пример) | Диплинк |
|---|---|---|
| Новый ежедневный пазл | «Новый пазл дня готов!» | `/daily` |
| Колесо доступно | «Крутани колесо — бесплатный спин» | `/wheel` |
| Риск потери стрика | «Не теряй серию из N дней» | `/daily` |
| Возврат D1/D3/D7 | «Тебя ждут новые слова» | `/` |
| Старт события | «Турнир начался — успей в топ» | `/events` |

- Ручные кампании — из консоли RuStore; триггерные — серверный планировщик (минимальный, по `lastActiveAt`).
- Частотный кап и тихие часы (конфиг). Уважать отказ (settings).

## Настройки

Тумблер «Уведомления» в `/settings` → включает/выключает локальную подписку и помечает opt-out на бэкенде.

## Тесты

Регистрация/дерегистрация токена, парс диплинков, soft-ask логика (показ один раз), уважение opt-out.
