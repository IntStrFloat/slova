# 13 — Производительность

Статус: утверждено · Обновлено: 2026-06-26

## Бюджеты

- Свайп по буквам и анимации — **60fps** на бюджетных Android (minSdk 23+).
- Холодный старт ≤ 3–4 c до интерактива карты.
- Стартовый бандл малый: картинки локаций и контент-паки уровней — **ленивая докачка по мирам** (expo-asset/expo-image), не в APK целиком.

## Правила жестов/анимаций

- Трекинг пальца по диску — **worklet** на UI-потоке; позиция/линия/подсветка — `shared values`. В drag-пути **нет** `setState`/`runOnJS` до завершения слова (onRelease).
- `engine.submitWord` — синхронный, дешёвый (работает с пулом уровня, не с мегасловарём [03]).
- Рендер сетки/диска — `react-native-svg` + Reanimated; избегать перерисовки всей сетки на каждое касание (мемоизация клеток, обновление только затронутых).

## Память/ассеты

- Словарь целиком в рантайм **не грузим** — только `answers`+`bonusPool` уровня.
- Изображения — кэш expo-image, освобождение при уходе из мира.
- Звуки — короткие, предзагрузка пула (expo-audio).

## Нативные зависимости (бюджет)

Сверх Expo допустимы: `react-native-mmkv`, `react-native-svg`, `yandex-mobile-ads`, RuStore billing/push. Новые native-deps — только с обоснованием здесь.

## Контроль

- Профилирование свайпа (нет JS-фреймдропов), холодный старт, размер AAB.
- Регресс-чек перед релизом: список тяжёлых экранов (карта с длинной дорогой, коллекция с картинками).
