# 20 — Design upgrade: приятность, конкурентность, polish

Статус: draft · Обновлено: 2026-06-29

## Цель

Довести визуальное и интерактивное качество клиента до уровня, при котором `slova`
ощущается не как типовая word-connect игра, а как премиальная word-travel puzzle:
приятнее в ежедневном использовании, богаче по game feel и аккуратнее в деталях,
чем прямые конкуренты жанра.

Критерий успеха: пользователь с первого экрана понимает, что это игра про слова,
прогресс и путешествие, а не набор одинаковых меню поверх красивого фона.

## Текущая база

Сильные стороны:

- Expo Router + React Native, единая дизайн-система в `src/ui/theme.ts`.
- Узнаваемый шрифт `Unbounded`, не системный UI-font.
- Фотофоны миров, travel-тема, коллекция достопримечательностей.
- Игровой экран уже содержит базовый game feel: жесты, линии выбора, haptics,
  audio feedback, success/error состояния, level complete overlay.
- Палитра `word-green + reward-gold` понятна: зелёный — действие, золото —
  награда.

Главные слабые места:

- Игровой экран на узком Android viewport обрезает ряд подсказок по краям и
  прижимает диск букв к нижней границе.
- Utility-экраны (`shop`, `settings`, `daily`, `profile`, `teams`, `events`)
  отличаются в основном текстом, а не композицией или поведением.
- Home сейчас ближе к меню-плиткам, чем к карте прогрессии/путешествия.
- В социальных экранах видны битые имена (`����`), что разрушает ощущение
  production quality.
- Недостаточно эмоциональных моментов: открытие места, прохождение уровня,
  дневная награда, колесо и лидерборд должны ощущаться как разные игровые
  события.

## Design direction

Направление: **premium travel word game**.

Тон:

- спокойный, взрослый, тактильный;
- без мультяшной кислотности и без “AI-gradient” эстетики;
- больше глубины, слоя, света и реакции на действие;
- интерфейс должен помогать игре, а не превращать её в админку.

Дифференциатор:

> Живой маршрут по городам и достопримечательностям, где каждое слово продвигает
> игрока по путешествию.

Не добавлять новый визуальный язык без необходимости. Усиливать текущий:
фотофон мира, стекло, золото награды, зелёные действия, крупные буквы.

## Глобальные правила UI

- Android-only: проверять в первую очередь 360x800, 393x852, 412x915 и малую
  высоту 360x740.
- Все ключевые touch targets: минимум 48dp, для основных действий 52-60dp.
- Ничего интерактивного не должно обрезаться safe-area, баннером рекламы или
  краями viewport.
- `AppText` остаётся обязательным для текста. Сырые `Text` допускаются только
  внутри сторонних/низкоуровневых компонентов при невозможности использовать
  `AppText`.
- Цвета только из `src/ui/theme.ts`; новые цвета сначала добавляются как токены
  с понятной ролью.
- Избегать вложенных карточек. Карточка внутри карточки допустима только для
  маленьких повторяемых items, но не для целых секций.
- У каждого экрана должен быть один главный visual anchor: карта, диск, витрина,
  таблица, колесо, календарь, профиль. Не превращать всё в одинаковые panels.
- Состояния loading/offline/empty/error должны быть визуально спроектированы, а
  не показаны строкой текста.

## Priority 0 — блокеры качества

### Игровой экран: адаптивность

Проблема: на текущем скриншоте подсказки обрезаются по горизонтали, диск букв
упирается в низ.

Требования:

- `LetterDisk` получает размер от доступной ширины/высоты, а не фиксированный
  `SIZE = 300` для всех экранов.
- Ряд подсказок не должен выходить за экран. Варианты:
  - компактный `2 + 1` layout;
  - горизонтальный scroll без обрезки;
  - floating radial/toolbar controls вокруг диска;
  - две иконки + primary подсказка, если высоты мало.
- На малой высоте gameplay приоритетнее bonus tray. Bonus tray должен уметь
  сворачиваться в компактный badge.
- Минимальный нижний отступ до safe-area/ad area: 12dp.

Acceptance:

- На 360x800 все три подсказки доступны, читаемы и не обрезаны.
- На 360x740 диск, текущая word-pill и минимум одна строка сетки остаются
  полностью видимыми.
- Swipe по буквам не конфликтует с нижней системной навигацией Android.

### Кодировка и имена игроков

Проблема: `����` в leaderboard/events/teams визуально воспринимается как
сломанная игра.

Требования:

- Не показывать битую строку как nickname.
- Если сервер/кэш возвращает некорректные символы, заменить на fallback вида
  `Игрок ####`.
- Для слишком длинных имён использовать `numberOfLines={1}` + ellipsis, но
  сохранять tag/score.

Acceptance:

- В leaderboard/events/teams не видно replacement characters.
- Длинные имена не вытесняют score за край строки.

## Priority 1 — core game feel

### CrosswordGrid

Текущая сетка читаема, но пустые клетки слишком “призрачные”.

Требования:

- Пустые клетки должны быть заметнее на сложном фотофоне: выше opacity или
  отдельный token `gridEmptyStrong`.
- При найденном слове буквы появляются последовательно по клеткам, а не просто
  через общий pulse.
- Подсказка буквы/слова должна иметь отдельное визуальное состояние:
  brief glow + amber outline.
- Для длинных сеток grid должен масштабироваться от контейнера, не ломая
  пропорции и не перекрывая bonus tray.

Acceptance:

- Игрок без усилий видит форму кроссворда на фоне Парижа/Нью-Йорка/Москвы.
- Success-анимация длится 250-450ms и не блокирует следующий ввод.

### LetterDisk

Текущий диск функционален, но ему не хватает “дорогой” тактильности.

Требования:

- Диск масштабируется: базовый размер 280-320dp, но уменьшается на низкой высоте.
- Активная буква получает не только amber fill, но и небольшое поднятие/свечение.
- Линия связи должна проходить под буквами, не перекрывая читаемость glyphs.
- Ошибка трясёт word-pill/линию, но не весь диск.
- Shuffle даёт короткую перестройку букв, а не только pulse.

Acceptance:

- Swipe ощущается мгновенным; setState/runOnJS не создаёт заметной задержки.
- При быстром вводе feedback не накапливает конфликтующие анимации.

### Level complete

Текущий overlay понятен, но недостаточно запоминается.

Требования:

- Сначала завершение сетки, затем reward moment.
- Монеты должны визуально “прилетать” в HUD или показывать короткий count-up.
- `double reward` должен быть вторичным, но привлекательным: gold CTA, clear
  rewarded icon, без давления.
- Если открыт landmark, переход от level complete к reveal должен ощущаться как
  единая цепочка награды.

Acceptance:

- Игрок видит, за что получил награду: уровень, бонусные слова, daily reward.
- Невозможно дважды начислить награду быстрыми тапами.

## Priority 2 — home as world map

Home должен быть не сеткой функций, а главным игровым маршрутом.

Требования:

- Основной экран: вертикальная карта/дорога уровней текущего мира.
- Текущий уровень — самый заметный узел с CTA `Играть`.
- Пройденные уровни, текущий уровень, locked/future уровни визуально различимы.
- Достопримечательности должны быть видны как milestone на маршруте.
- Daily/wheel/events/shop/profile/settings становятся secondary actions:
  compact top/bottom shortcuts, badges, или drawer-like menu.
- Coin HUD остаётся в top-right.

Acceptance:

- Первый viewport показывает текущий уровень, следующий milestone и хотя бы
  намёк на дальнейший путь.
- Пользователь может начать игру одним тапом.
- Home не выглядит как “9 одинаковых кнопок”.

## Priority 3 — distinct screen identities

### Daily

Тон: ритуал и возвращение.

Требования:

- Calendar reward должен быть не сеткой маленьких квадратов, а короткой
  progression strip: вчера/сегодня/завтра/7-й день.
- Streak — визуальный якорь, не просто `🔥 0`.
- Claimed state: спокойная награда + next reset hint.

### Wheel

Тон: яркий reward moment.

Требования:

- Колесо должно выглядеть богаче: центральный hub, разделители, pointer shadow,
  итоговый сектор подсвечивается после остановки.
- Кнопка rewarded spin disabled-state должна быть настоящим disabled, не только
  opacity.
- Результат — modal/toast с count-up монет.

### Collection

Тон: музей/альбом путешествий.

Требования:

- Закрытые карточки не должны выглядеть как broken image placeholder.
- Locked landmarks: силуэт/blur/dimmed photo, подпись `???`, прогресс до
  открытия.
- Открытая карточка: крупное фото, страна/мир, короткий факт.
- Reveal overlay должен быть hero-моментом с фото как главным объектом, а не
  маленькой картинкой внутри панели.

### Shop

Тон: витрина, не список настроек.

Требования:

- Packs различаются визуально: small/medium/best value.
- Лучшее предложение получает badge, но без агрессивного dark pattern.
- `Remove ads` и `Pro` отделены от coin packs.
- Цены не должны сжимать или ломать русские подписи.

### Leaderboard / Events

Тон: соревнование.

Требования:

- Top-3 визуально отделены: медали/подиум/акцентные строки.
- Строка игрока всегда заметна, даже если он вне top viewport.
- Weekly/global tabs должны быть segmented control, а не две обычные CTA-кнопки.
- Offline cache state не должен выглядеть как ошибка игры; показывать subtle
  banner.

### Teams

Тон: клуб.

Требования:

- Empty state должен мотивировать создать/вступить, а не просто показывать форму.
- Team card: эмблема, участники, score, CTA.
- Поле ввода должно иметь тот же шрифт/цветовой язык, что `AppText`.

### Profile / Settings

Тон: calm utility.

Требования:

- Profile: avatar/initial badge, nickname, level progress, collection progress.
- Settings: компактные группы, icons for sound/music/haptics/privacy, версии и
  restore purchase как secondary section.
- Switch rows не должны занимать визуальный вес игрового контента.

## Motion contract

Использовать Reanimated для high-impact моментов:

- page enter: лёгкий stagger для главного anchor и CTA;
- word found: sequential grid fill;
- invalid: короткая shake-анимация только feedback layer;
- reward: count-up + coin movement;
- landmark reveal: image scale/fade + title reveal;
- wheel: spin easing + settle bounce.

Ограничения:

- Drag path у `LetterDisk` должен оставаться быстрым.
- Не добавлять бесконечные декоративные анимации на utility-экраны.
- Уважать будущий `reduced motion` setting, если он будет добавлен.

## Accessibility / readability

- Контраст текста на фотофонах: минимум WCAG AA для основного текста.
- Минимальный body text: 15sp, важные CTA 15-17sp.
- Для русского текста избегать узких контейнеров, где слова ломают layout.
- Все icon-only controls должны иметь accessibility label.
- Disabled actions должны иметь `disabled` semantics, не только opacity.

## Visual QA

Перед принятием UI-изменений собрать скриншоты:

- `/onboarding`
- `/`
- `/game`
- `/daily`
- `/wheel`
- `/collection`
- `/shop`
- `/leaderboard`
- `/events`
- `/teams`
- `/profile`
- `/settings`

Viewport set:

- 360x740
- 360x800
- 393x852
- 412x915

Проверить:

- нет обрезанных кнопок/текста;
- нет replacement characters `�`;
- основные CTA видны без скролла там, где это критично;
- safe-area и ad banner не перекрывают gameplay;
- фон не мешает чтению текста;
- каждый экран имеет свой visual anchor.

## Implementation order

1. Исправить P0: game адаптивность + битые nickname fallback.
2. Усилить core game feel: grid reveal, disk scaling, reward moment.
3. Переделать home в карту прогрессии.
4. Развести identities для daily/wheel/collection/shop/leaderboard/teams.
5. Финальный pass дизайн-системы: вынести повторяемые паттерны в primitives,
   убрать одноразовые inline-стили там, где они стали общими.

## Non-goals

- Не добавлять iOS-специфичные паттерны.
- Не менять backend/API только ради визуального редизайна, кроме безопасной
  клиентской обработки некорректных имён.
- Не добавлять новые зависимости без отдельного решения.
- Не делать маркетинговую landing page внутри приложения.

## Тесты и гейты

- `npm run typecheck`
- `npm test`
- `npm run lint`
- ручной или автоматический screenshot pass по viewport set выше;
- визуальная проверка gameplay на Android/emulator: swipe, hints, shuffle,
  level complete, landmark reveal.
